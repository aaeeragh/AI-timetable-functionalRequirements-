Automatic Timetable Generation System

Functional Requirements

This system automates the creation of academic timetables, ensuring conflict-free schedules for students, professors, and rooms. Below are the core functional requirements:

1. User Authentication

  • Requirement: Only authorized personnel (admin, faculty) can access the timetable generation features.
  • Description: Secure login mechanisms will be implemented to ensure only valid users can generate or edit timetables.
  • Process Flow: User enters credentials → System validates → Access granted on successful validation.

2. Input Subject and Staff Information

  • Requirement: The admin must input details about subjects, faculty, and classrooms.
  • Description: The system will require detailed information about subjects, faculty availability, and classroom resources.
  • Process Flow: Data entered → System stores subject, faculty, and room info.

3. Input Constraints

  • Requirement: Faculty members and rooms may have availability constraints that must be considered.
  • Description: Constraints like unavailable time slots or room bookings must be entered into the system.
  • Process Flow: User specifies constraints → System respects constraints during timetable generation.

4. Generate Timetable

  • Requirement: The system automatically generates a conflict-free timetable.
  • Description: Based on inputs and constraints, the system generates a draft timetable.
  • Process Flow: User requests generation → System processes → Draft timetable created.

5. Timetable Review and Editing

  • Requirement: The generated timetable can be reviewed and manually adjusted by the admin.
  • Description: Minor edits to the draft timetable can be made for optimization.
  • Process Flow: Review draft → Manual edits → Save and publish final timetable.

6. Publish Timetable

  • Requirement: The timetable will be accessible to students and faculty after publishing.
  • Description: The finalized timetable is published on a web interface for students and staff to view.
  • Process Flow: Admin publishes → Timetable available for users.
